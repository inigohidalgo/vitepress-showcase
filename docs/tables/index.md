# Table catalog

A tiny catalog of three tables, each page generated by the same `[table].paths.ts` loader. Add an entry to `_catalog.ts` and a new page appears on the next build.

<ul>
  <li v-for="t in tables" :key="t.name">
    <a :href="`./${t.name}`"><code>{{ t.schema }}.{{ t.name }}</code></a>
    — {{ t.description }}
  </li>
</ul>

## How this works

Three files in `docs/tables/`:

| File | Role |
| --- | --- |
| `_catalog.ts` | The data. Imagine a SQL query, a YAML file, a Unity Catalog dump. |
| `[table].paths.ts` | Exports `paths()`; returns one entry per generated page. |
| `[table].md` | Template; `$params` holds per-page data, `<!-- @content -->` injects the loader's content string. |
